## 核心概念与标准

- 读题时先抓住 **命令词**：evaluate（评价好坏）、outline（概述现状）、comment on strengths（优点）、make suggestions for improvements（改进建议），这些就是你正文必须一一覆盖的功能，否则内容分会丢分。
- evaluate 的内核是“给东西定价值”：说清楚哪些好、哪些不好，而不是单纯描述。
- 内容想点子时，从具体设施出发：地铁/公交/自行车道/步行友好度等，宁可少点子但要具体细节。
- 四大评分维度：
    - Language：语法+词汇要有一定难度和准确度，尽量用和话题相关的精准词（如 pedestrian-friendly）。
    - Organization：清晰的段落结构，报告/提案强烈建议用 **标题 + 小标题 + 可能的 bullet points**。
    - Communicative achievement：语气客观、事实性、清晰，符合“给上级/老师写报告”的场景。
    - Content：紧扣题目要求，每一项任务点都有具体细节支撑。
- 报告/提案的读者一般是“老师/老板/主办方/负责人”这类更高位的人，语域要正式、保持一定距离、避免 I/you，多用被动和第三人称。

## 报告 vs 提案：结构性区别

- 报告和提案的共同点：
    - 都是正式、结构清晰的文体，主要任务是向更重要的决策者呈现信息并提出建议。
- 关键区别（负载信息）：
    - 报告更偏向“**现在的情况如何**”：描述现状 + 适度建议。
    - 提案更偏向“**未来应该怎么做**”：强调新的想法、改进方案和你的立场。
- 同一道题如何判断是 report 还是 proposal：
    - 如果题目在描述已有系统、要求你“report on”某个现有状况 → 报告为主。
    - 如果题目在谈“计划拆除某建筑/推出新课程/改变某制度”，并让你说“why should it be preserved / what could be done” → 提案为主，重在未来行动。

## 通用写作框架（包含行动步骤）

## 1. 通用规划原则

- **永远先计划，再写**：老师建议“计划时间几乎和写作时间一样长”，例如 10–15 分钟规划，写起来会更快更符合标准。
- 计划时要同时解决：
    - 要覆盖哪些任务点（命令词拆分）。
    - 读者是谁、我是什么角色。
    - 每个段落的小标题是什么，每段要给出至少 1–2 个具体例子。

## 2. 报告模板：一步步可照做

以“城市公共交通”报告为例。

1. 标题
    
    - 模板：Report on + 话题（Report on Public Transport in Seville）。标题可以“无聊但清楚”。
        
2. 引言（说明目的）
    
    - 功能：一句话告诉读者“这份报告要做什么”。
    - 模板动作：
        - 用结构：This report aims to + 命令词（evaluate / outline / comment on / suggest）。
            
3. 现状段落（outline）
    
    - 每个子段对应一个系统或方面：如 Subway，Tram，Cycling。
    - 每段至少包括：
        - 描述：它现在是怎样的（线路、速度、舒适度等）。
        - 简要价值判断：谁受益、谁不方便。
            
4. 优点 & 缺点段落（evaluate & strengths）
    
    - 在描述中顺带完成评价：例如“地铁现代快速，但只有一条线，多数人用不上；有轨电车环保，但速度太慢且和地铁线路重叠”。
        
5. 改进建议段落（suggestions）
    
    - 懒人对齐法：让建议直接对应你前面指出的每一条问题。
        - 问题：只有一条地铁线 → 建议：增设南北向线路。
        - 问题：电车线路和地铁重叠 → 建议：延伸到机场/火车站等关键节点。
            
6. 结尾（总结 + 温和立场）
    
    - 功能：用正式表达把“你应该做这些”说出来。
    - 模板动作：
        - Should these suggestions be implemented, ...（如果采纳这些建议，将会……）。

## 3. 提案模板：一步步可照做

以“拯救一栋旧楼”的提案为例。

1. 标题
    
    - Proposal to Preserve + 建筑名称（自己起名字，让场景更真实）。
        
2. 引言：交代对象 +目的
    
    - 动作：先说这是什么建筑（旧图书馆/体育中心）、在哪条街、目前状况（废弃、将被拆除）。
    - 然后用一句常用结构：This proposal aims to explain why the building should be preserved and suggest ways in which it could be modernised and used.
        
3. “为什么值得保留”段落
    
    - 行动步骤：至少给出 2–3 个角度：
        - 历史价值/建筑风格。
        - 对当地人的情感或集体记忆。
        - 地理位置优势。
            
4. “如何现代化/改造”段落
    
    - 明确用途（youth club 比 nightclub 更合适考试语境）。
    - 具体化改造措施：改变功能、翻新内部、增加科技设备、提高能效等。
        
5. “对谁有何好处”段落
    
    - 直接回答题目里的 “how it would benefit local people”。
    - 具体对象 + 具体收益：当地青年可以在此参加课程、运动、社交；带动附近商铺等。
        
6. 结尾：重申立场
    
    - 用正式表达重复你的主张：I strongly propose that the building (name) should be preserved and converted into ...

## 必须掌握的句型与改写思路（负载类语言点）

- 老师强调：不要“背一串句子”，要学会把它们拆成“核心意思 + 可替换部分”，自己改写。
- 典型开头结构：
    - The aim of this proposal is to put forward a piece of academic research that I would like to undertake.
        - 核心意思：我打算做一个项目/计划。
        - 可操作改写：
            - The goal/objective of this proposal is to present/outline a project/plan that I would like to carry out.
- 说明将要讨论什么的结构：
    - This proposal outlines the scope of the new course, explaining why it would be of benefit and exploring how it could be productively carried out.
        - 可替换：outlines → sketches/summarises/introduces；benefit → be beneficial/advantageous；用被动替换 I：Some ideas are offered ...
- 表明写作原因的结构：
    - There are several factors which have led to the submitting of this proposal.（多点原因、较正式）。
- 论证用语模板：
    - There are several convincing/logical factors that support this idea.
    - To begin with / Most importantly, it would foster/promote/stimulate ...
    - What is more / Furthermore / Additionally, it could result in / bring about ...
- 作业/训练动作：
    - 对每个“简单想法”（例如：虽然贵但值得）自己写出 1–2 个正式表达，要求：
        - 用抽象名词（benefits, advantages, drawbacks）。
        - 多用被动和名词化结构（The cost is justified by ...）。

## 模型答案里体现出来的“隐藏动作”

老师自己写的在线研讨会报告，反向拆出来是这样用的：

- 内容层：
    - 明确勾选所有任务点：经历概述 + strengths + weaknesses + suggestions，且每一项都有具体例子（软件卡顿、不能交流等 → 建议更好平台和 breakout rooms）。
- 语气和读者意识：
    - 假定组织者是“大人物（big cheeses）”，所以用 I would like to offer some feedback 这类礼貌、间接的说法，而不是 You should change this。
- 组织层：
    - 明确标题、小标题，对应任务点；
    - 使用清晰的引用结构和连接词：relative clauses（which aim to…）、in every session、because it was online、more importantly 等。
- 语言层：
    - 有意识地加入“高价值但自然”的小点：
        - having done this, I would …
        - to give advice on…，to take place，experts in their fields，enthusiastically，at once，one of the highlights，slightly varied，peers 等。
    - 写完后再进行“语言加强”：在普通的句子里补一个更具体的副词/形容词（如加 enthusiastically），让整体档次抬一档。

## 最后总结成可执行清单

下次你写 C1 报告/提案时，可以直接按这张 checklist 操作：

- 读题：圈出所有命令词（evaluate/outline/comment/suggest 等），每个命令词对应一个或多个段落。
- 定位类型：看任务是描述现状还是主意/未来行动，决定是 report 还是 proposal。
- 角色与读者：明确“我是谁、写给谁”，统一用正式语气，被动+第三人称，避免直接 you。
- 计划结构（10–15 分钟）：
    - 标题：Report on / Proposal to ...
    - 引言：一句话写明目的（This report aims to… / This proposal aims to…）。
    - 2–3 个主体段落：每段一个方面，小标题清晰，包含现状 + 评价 + 具体例子。
    - 建议段落：让每条建议直接“修补”前文提到的缺点。
    - 结尾：一句条件句或总结性句式重申为什么应该采纳建议（Should these suggestions be implemented, ...）。
- 语言升级：
    - 每段至少加 1 个专业/话题词（如 pedestrian-friendly, peer, video calls）。
    - 每篇至少用 2–3 个高阶连接词和结构（having done..., which aim to..., furthermore, more importantly）。
    - 写完后专门用 2–3 分钟做“词汇加料”和错误检查。
